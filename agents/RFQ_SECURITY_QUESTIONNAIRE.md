## بخش ۲: انطباق با معماری جداسازی مغز/بدن (ماده ۲ Annex 13)

| سؤال | پاسخ (بله/خیر/توضیح) |
|-------|----------------------|
| آیا محصول شما **بدون ابر اختصاصی** قابلیت کار دارد؟ | |
| آیا الگوریتم‌های DLM یا بهینه‌سازی انرژی در فریمور دستگاه پیاده‌سازی شده است؟ | |
| آیا دستگاه فقط **دستورات امضا شده** از سمت ابر را اجرا می‌کند؟ | |
| آیا امکان **غیرفعال‌سازی کامل ابر موازی** (در صورت وجود) وجود دارد؟ | |

---

## بخش ۳: انطباق با قفل رمزنگاری (ماده ۳ Annex 13)

| سؤال | پاسخ |
|-------|------|
| آیا دستگاه از **Secure Boot** (بوت امن) پشتیبانی می‌کند؟ | |
| آیا فلش مموری دستگاه **به‌صورت کامل رمزگذاری** شده است (Flash Encryption)؟ | |
| آیا دستگاه دارای **تراشه امن (Secure Element - SE)** مجزا از پردازنده اصلی است؟ | |
| آیا کلیدهای رمزنگاری در SE ذخیره می‌شوند و غیرقابل استخراج هستند؟ | |
| مدل/سری تراشه امن استفاده‌شده چیست؟ | |

---

## بخش ۴: انطباق با Dark Provisioning (ماده ۴ Annex 13)

| سؤال | پاسخ |
|-------|------|
| آیا دستگاه‌ها را به‌صورت **خام (Bare Metal)** بدون هیچ کلید یا فریمور عملیاتی تحویل می‌دهید؟ | |
| آیا اجازه می‌دهید **فرآیند تزریق کلید (Provisioning)** توسط تیم زینوا در مرکز خودمان انجام شود؟ | |
| آیا دسترسی **Root** به دستگاه برای تأمین‌کننده محفوظ است؟ (پاسخ باید "خیر" باشد) | |

---

## بخش ۵: انطباق با Kill-Switch (ماده ۶ Annex 13)

| سؤال | پاسخ |
|-------|------|
| آیا دستگاه قابلیت **غیرفعال‌سازی کامل از راه دور (Kill Switch)** از طریق ابر زینوا را دارد؟ | |
| آیا این قابلیت در سطح **سخت‌افزار** (غیرقابل دور زدن با فلش مجدد) پیاده‌سازی شده است؟ | |

---

## بخش ۶: انطباق با Zero Trust (ماده ۷ Annex 13)

| سؤال | پاسخ |
|-------|------|
| آیا تمام ارتباطات (OCPP, MQTT, REST) از **TLS 1.3** پشتیبانی می‌کنند؟ | |
| آیا **احراز هویت متقابل (mTLS)** برای تمام اندپوینت‌ها پشتیبانی می‌شود؟ | |
