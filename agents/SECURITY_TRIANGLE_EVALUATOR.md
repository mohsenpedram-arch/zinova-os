## ۱. ساختار مثلث امنیتی

| رأس | عنوان | وزن (٪) | شاخص‌های کلیدی |
|-----|-------|---------|---------------|
| **رأس A** | حاکمیت فنی (Sovereignty) | **۵۰٪** | پذیرش Dark Provisioning، عدم تحمیل ابر، Kill Switch، عدم دسترسی Root |
| **رأس B** | یکپارچگی سایبر-فیزیک (Integrity) | **۳۰٪** | Secure Boot، Flash Encryption، Secure Element، TLS 1.3/mTLS |
| **رأس C** | تاب‌آوری زنجیره تأمین (Resilience) | **۲۰٪** | گواهینامه‌ها، ظرفیت تولید، زمان تحویل، هزینه، سابقه همکاری |

---

## ۲. روش امتیازدهی (هر رأس از ۰ تا ۱۰)

### رأس A (حاکمیت - وزن ۵۰٪)
- ۱۰ = تمام موارد (Bare Metal، بدون ابر، Kill Switch HW، عدم دسترسی Root) تأیید شد.
- ۵ = بیشتر موارد تأیید شد، اما یک نقص جزئی (مثلاً Kill Switch نرم‌افزاری).
- ۰ = ابر اختصاصی تحمیل می‌شود یا دسترسی Root به تأمین‌کننده داده می‌شود. **رد فوری**.

### رأس B (یکپارچگی - وزن ۳۰٪)
- ۱۰ = Secure Boot + Flash Encryption + SE (با مستندات کامل).
- ۵ = فقط Secure Boot و Flash Encryption (بدون SE).
- ۰ = هیچ‌کدام (قابل قبول نیست مگر برای نمونه‌های اولیه آزمایشگاهی).

### رأس C (تاب‌آوری - وزن ۲۰٪)
- ۱۰ = CE/RoHS/TUV/ISO، ظرفیت ۱۰۰۰+، تحویل < ۴ هفته، قیمت رقابتی.
- ۵ = گواهی محدود، ظرفیت ۵۰۰-۱۰۰۰.
- ۰ = بدون گواهی یا تحویل > ۸ هفته.

## ۳. فرمول نهایی امتیاز

**امتیاز نهایی = (امتیاز A × ۰.۵) + (امتیاز B × ۰.۳) + (امتیاز C × ۰.۲)**

**شرط ورود به مرحله نهایی:**  
- امتیاز A نباید کمتر از ۷ باشد (در غیر این صورت، حتی با نمرات بالا در سایر بخش‌ها، تأمین‌کننده رد می‌شود).
